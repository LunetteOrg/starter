# ADR-0015: Storybook as the showcase for `packages/ui`

- Status: accepted
- Date: 2026-06-29

## Context

Shared components live in `packages/ui` and tokens in `packages/ui-tokens` (ADR-0014). We need a place to develop them in isolation, document them and check their accessibility, without coupling this showcase to a domain app (the starter is a template: apps stay generic).

## Decision

Storybook lives in `apps/design-system` as a **generic tooling app**, on `@storybook/react-vite` (Storybook 10, Vite).

- **Stories are co-located** in `packages/ui` (`packages/ui/src/**/*.stories.tsx`); the design-system app aggregates them via glob — adding a component with its story makes it show up with no extra configuration.
- The **Foundations** (tokens) are generated by `storybook-design-token`, reading the `@tokens` annotations of `@starter/ui-tokens` (ADR-0014); values are not duplicated in the docs.
- **A11y**: `@storybook/addon-a11y` with `a11y.test: 'todo'` — axe violations are reported but non-blocking; switch to `'error'` to make them blocking.
- `reactDocgen: 'react-docgen-typescript'` documents component props automatically.
- `pnpm --filter @starter/design-system build` produces the static Storybook in `build/` (Turbo cache, runs in the `build` pipeline).

```ts
// apps/design-system/.storybook/main.ts
stories: [
  '../src/**/*.mdx',
  '../../../packages/ui/src/**/*.stories.@(ts|tsx)',
]
```

## Consequences

- \+ Components are developed in isolation; stories sit next to the code they document.
- \+ The Storybook build runs in CI: a broken story or a missing import fails the build.
- \+ The showcase is decoupled from domain apps: it stays valid in every project derived from the starter.
- − One extra heavy dependency (Storybook + Vite) in the monorepo, just for dev/docs.
- − The "story next to the component in `packages/ui`" convention must be respected, otherwise stories aren't aggregated.
